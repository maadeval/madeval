Librerias de componentes como [Chakra UI](https://chakra-ui.com/) utilizan en muchos de sus componentes una propiedad llamada `as` que permite renderizar componentes con un tag HTML distinto al que tiene por defecto.

Por ejemplo, el componente `Button` de Chakra UI tiene por defecto un tag `button`, pero si le pasamos la propiedad `as={"a"}` entonces se renderizar치 de esa manera.

```jsx title="App.jsx"
const Button = (props) => <button {...props}>Guardar cambios</button>

// 游녤 se renderiza en el DOM como: <button>Guardar cambios</button>

const Button = (props) => (
  <button as="a" {...props}>
    Guardar cambios
  </button>
)

// 游녤 se renderiza en el DOM como: <a>Guardar cambios</a>
```

Esto suele ser muy util para componentes como `Text` y `Heading`, en el que podemos renderizar un `h1`, `h2`, `h3`, etc. dependiendo del valor que le pasemos.

```jsx title="App.jsx"

// 游녤 se renderiza en el DOM como: <h1>T칤tulo</h1>

  <h1 as="h2" {...props}>
    T칤tulo
  </h1>

// 游녤 se renderiza en el DOM como: <h2>T칤tulo</h2>
```

Ahora bien, esto no solo queda aqui! No todas las etiquetas HTML comparten los mismos atributos, por ejemplo, un `button` tiene el atributo `type` pero un `a` no, por otra parte un `a` tiene el atributo `href` pero un `button` no.

Seria interesante cambiar los atributos que permite un componente en base al tag que le pasemos.
